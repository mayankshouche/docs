Le menu de navigation est la liste des liens sur chaque site web.

Vous devrez probablement mettre à jour `docs.json` chaque fois que vous ajoutez une nouvelle page. Les pages n'apparaissent pas automatiquement.

## Syntaxe de navigation

Notre syntaxe de navigation est récursive, ce qui signifie que vous pouvez créer des groupes de navigation imbriqués. Vous n'avez pas besoin d'inclure `.mdx` dans les noms de pages.

<CodeGroup>

```json Navigation Régulière
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Pour Commencer",
          "pages": ["quickstart"]
        }
      ]
    }
  ]
}
```

```json Navigation Imbriquée
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Pour Commencer",
          "pages": [
            "quickstart",
            {
              "group": "Pages de Référence Imbriquées",
              "pages": ["page-reference-imbriquee"]
            }
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

## Dossiers

Mettez simplement vos fichiers MDX dans des dossiers et mettez à jour les chemins dans `docs.json`.

Par exemple, pour avoir une page à `https://votresite.com/votre-dossier/votre-page`, vous créeriez un dossier appelé `votre-dossier` contenant un fichier MDX appelé `votre-page.mdx`.

<Warning>

Vous ne pouvez pas utiliser `api` comme nom de dossier sauf si vous le placez dans un autre dossier. Mintlify utilise Next.js qui réserve le dossier `api` de premier niveau pour les appels serveur internes. Un nom de dossier comme `api-reference` serait accepté.

</Warning>

```json Navigation Avec Dossier
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Nom du Groupe",
          "pages": ["votre-dossier/votre-page"]
        }
      ]
    }
  ]
}
```

## Pages Cachées

Les fichiers MDX non inclus dans `docs.json` n'apparaîtront pas dans la barre latérale mais sont accessibles via la barre de recherche et en les liant directement.
